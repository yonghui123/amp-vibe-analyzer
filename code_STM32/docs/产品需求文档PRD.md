# 旋转设备电流与振动信号采集分析系统 — 产品需求文档 (PRD)

> **文档版本**：V1.0  
> **产品线**：旋转设备状态监测系统  
> **适用平台**：嵌入式终端（STM32F407VGT6 + 7寸 RA8875 触摸屏）  
> **编写角色**：产品经理

---

## 1. 项目背景与目标

### 1.1 业务背景

旋转设备（电机、轴承、泵体等）在工业生产中广泛应用，其运行状态直接影响产线效率与安全。目前主要依赖人工巡检，存在以下痛点：

- 故障发现滞后，非计划停机造成产能损失
- 缺乏量化数据支撑，维修决策依赖经验
- 无法实现7×24小时连续监测

### 1.2 产品定位

面向设备维护工程师和产线管理者，提供**嵌入式实时采集 + 上位机数据分析**的一体化解决方案，实现：

| 维度 | 目标 |
|------|------|
| 采集 | 多通道信号（通用 ≥ 2路 + 振动 ≥ 2路）实时同步采集 |
| 处理 | 滤波、包络计算、FFT频谱分析 |
| 预警 | 双级报警（通用超限报警 + 振动包络报警） |
| 管理 | 历史数据查询、趋势分析、报表导出 |

### 1.3 核心用户场景

1. **日常监测**：设备维护工程师在主画面观察实时信号曲线，确认设备运行正常
2. **异常响应**：系统触发报警后，工程师通过弹窗快速定位异常通道，确认后复位
3. **故障分析**：通过历史曲线对比不同时段数据，定位故障发生时间节点
4. **参数调优**：根据设备型号调整采集频率、滤波方式、报警阈值等参数

## 2. 系统架构

本系统为**独立嵌入式终端设备**，所有采集、处理、显示、交互均在集成触摸屏上完成，无需外接PC。

│  │ (触摸交互)│  │ (触摸交互)│  │ (实时曲线)│  │ (触摸查询)   │ │

│  ┌──────────┐  ┌──────────┐  ┌──────────┐                   │

#### 3.1.2 通道配置表字段定义

| 字段 | 类型 | 通用通道规则 | 振动通道规则 | 校验规则 |
|------|------|------------|------------|---------|
| 通道号 | 只读 | 1~N自动编号 | 同左 | — |

| 灵敏度 | 浮点输入 | 不显示 | 0.1~100（2位小数），单位mV/μm | 仅振动通道 |

2. 切换通道类型时，右面板参数自动切换

### 3.2 模块二：采集设置 / 修整参数（参数控制层）

**开发优先级：P1**

#### 3.2.1 界面布局

- **右区**：参数说明文本

#### 3.2.2 参数分组定义

**分组一：周期与通道**

| 参数 | 规则 | 示例 |
|------|------|------|
| 修整最小循环时长 | 1~3600s（整数），控制包络计算周期 | 400s |
| 振动通道选择 | 下拉，关联已启用的振动通道 | 轴承振动CH1 |
| 通用通道选择 | 下拉，关联已启用的通用通道（用于触发振动包络） | 主电机电流1 |

**分组二：滤波设置**

| 参数 | 规则 | 示例 |
|------|------|------|
| 滤波方式 | 不滤波 / 平滑滤波 / 转速滤波(带通) | 不滤波 |
| 转速(RPM) | 100~100000（整数），用于计算中心频率 f0=RPM/60 | 1500 |
| 周期倍数 | 0~100（整数），数据量=周期倍数+1 | 20 |
| 滑动平均 | 0~10（整数），窗口大小=滑动平均+1 | 10 |

> 显示条件：滤波方式始终显示；转速仅转速滤波时显示；周期倍数和滑动平均在平滑滤波/转速滤波时显示

**分组三：阈值设置**

| 参数 | 规则 | 示例 |
|------|------|------|
| 通用阈值 | 通用信号超出包络线的触发量，无固定范围，与通用通道单位一致 | 0 |
| 阈值时间 | 通用信号达到阈值后延迟触发时长，0.1~10s（1位小数） | 0.5s |
| 报警阈值 | 振动峰峰值超出包络线的时间占比，0%~100%（整数） | 10% |

**分组四：包络线设置**

| 参数 | 规则 | 示例 |
|------|------|------|
| 包络线上限/下限 | 相对于基准值的百分比，整数 | 上限20%，下限10% |
| 包络线文件 | 可选，支持导入CSV格式自定义包络线文件 | D:\包络线1.csv |

**包络线文件说明**：

包络线文件为可选项，可传可不传。其作用如下：

1. **基准折线**：文件中包含 N 个数据点 (X, Y)，这些点连接形成一条基准折线
2. **上包络线**：每个点的 Y 值 × 上限百分比 = 上边界值，所有上边界点连接形成上包络线
3. **下包络线**：每个点的 Y 值 × 下限百分比 = 下边界值，所有下边界点连接形成下包络线
4. **有效区间**：若文件含 N 个点，则水平方向有效区间为 [0, N]，垂直方向由上下包络线围成

**与主监控画面的关系**：

- **无包络线文件时**：主监控画面仅显示实时峰峰值曲线，无包络边界
- **有包络线文件时**：主监控画面额外显示上下包络线边界，振动峰峰值在包络区间内为正常，超出则计入报警时间占比

```csv
# 包络线文件格式示例 (CSV)
# X轴序号,Y轴基准值
0,0.5
1,0.8
2,1.2
3,1.0
4,0.6
```

## 4. 报警逻辑设计

### 4.1 通用信号报警（一级报警）

```
实时采集 → 信号值 ≥ 通用报警阈值？
  ├─ 否 → 继续采集
  └─ 是 → 开始计时 → 持续时间 ≥ 报警延迟(默认1S)？
              ├─ 否 → 继续计时
              └─ 是 → 触发报警（状态栏红闪 + 弹窗 + 蜂鸣）
```

### 4.2 振动包络报警（二级报警）

```
实时采集 → 通用信号 ≥ 通用阈值？
  ├─ 否 → 继续
  └─ 是 → 持续 ≥ 阈值时间？
              ├─ 否 → 继续
              └─ 是 → 采集振动信号 → 计算峰峰值 → 生成包络线
                        → 超限时间占比 ≥ 报警阈值？
                            ├─ 否 → 继续下一循环
                            └─ 是 → 触发振动报警
```

### 4.3 双报警（同时触发）

- 弹窗合并显示两类报警信息
- 状态栏快速闪烁（2次/秒）
- 蜂鸣频率1次/秒（可手动关闭）

## 5. 核心算法需求

### 5.1 峰峰值计算

| 步骤 | 说明 |
|------|------|
| 输入 | 振动原始信号（采样频率f，时长T=修整最小循环时长） |
| 分窗 | 按"周期倍数"划分时间窗，窗口数=周期倍数+1，窗口时长=T/(周期倍数+1) |
| 计算 | 每窗口取Max和Min，峰峰值=Max-Min |
| 输出 | 峰峰值数据序列 |

### 5.2 滤波算法

| 方式 | 实现 | 参数 |
|------|------|------|
| 不滤波 | 直接输出原始信号 | 无 |
| 平滑滤波 | 滑动平均：窗口内求和/窗口大小 | 滑动平均、周期倍数 |
| 转速滤波(带通) | f0=RPM/60，带通范围f0×(1±范围%)，IIR滤波器 | 转速(RPM) |

### 5.3 包络线生成

| 步骤 | 说明 |
|------|------|
| 输入 | 峰峰值数据序列（N个点） |
| 提取极值 | 遍历序列，提取局部最大值（上包络点）和局部最小值（下包络点） |
| 插值拟合 | 三次样条插值连接极值点 |
| 输出 | 与峰峰值曲线时间轴对齐的上/下包络线 |

| I2C EEPROM | 16KB (AT24C128) | I2C | 设备参数、校准数据、系统配置 |
