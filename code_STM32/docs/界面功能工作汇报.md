# SignalX 界面功能工作汇报

> **项目**：`code_STM32`（STM32F407 + RA8875 7 寸触摸屏 + LVGL v8.4）  
> **汇报日期**：2026-07-05  
> **说明**：本文档以 **板端实机** 为口径，汇报界面布局、触摸交互及与采集链路的对接情况；涉及文件存储的功能，在 SD 卡驱动未接入前一律视为未完成。

---

## 一、总体进展

系统采用 **四个 Tab 页** 组织主要功能：**通道设置 → 采集设置 → 主监控 → 报警记录**。四个界面的 **布局、控件与触摸交互框架已基本完成**；参数可在界面内编辑并作用于运行时逻辑；主监控已对接八通道采集任务，支持 Start/Stop 与实时曲线刷新；报警判定与弹窗逻辑已在软件层实现。

与产品需求文档（PRD）的完整目标相比，**依赖 SD 卡/Flash 的持久化与文件读写功能均未在板端闭环**，部分展示细节（数据面板、曲线标注等）尚未实现，**触摸校准、LVGL 内存、板端实机验收** 等硬件相关事项仍待后续协作推进。

## 三、遗留问题

当前主要遗留问题如下 **四项**：

**1. 电阻屏触摸点校准**

板端已实现五点仿射校准向导（`gui_calib.c`），但 **实际触摸落点与显示坐标仍存在偏差**，已尝试多种校准与映射方案，问题尚未完全解决。该问题与 RA8875 内置触摸 ADC、硬件接线及驱动时序密切相关。

**2. LVGL 内存与刷新性能**

LVGL 堆位于 CCMRAM（64KB），主监控 chart 点数、Tab 懒加载、50Hz 异步推送等已做初步优化，但 **板端仍可能存在内存偏紧、交互时刷新卡顿** 等情况。折线图时间窗目前仅约 2 秒，进一步扩展需结合 RAM 布局与刷新策略继续优化。

**3. SD 卡硬件已有、软件未接入**

板级已预留 SD 卡，但固件侧 **尚未完成 SD 卡驱动与 FatFs 文件系统集成**。因此包络 CSV 导入、配置/采集参数持久化、报警记录存储、采集过程自动存盘、历史数据查询与导出等功能均 **无法在板端闭环**。

**4. 板端实机联调与稳定性验收未完成**

**板端八通道 AD7606 真实采样、50Hz UI 推送稳定性、长时间运行（如 30 分钟无崩溃）** 等测试项尚未完成验收。部分功能（如曲线条数热更新、异常边界场景）也需在实机上进一步验证。

---

## 四、后续建议

上述遗留问题 **1～4 均涉及硬件平台、存储接口或实机验证**，单靠界面层软件难以独立闭环。建议后续安排 **熟悉本板硬件（RA8875 显示/触摸、FSMC 总线、SD 卡、AD7606 采集）的工程师** 参与联调：优先推进触摸校准与 SD 卡驱动，在此基础上再完成 LVGL 内存调优及板端整体验收；数据面板、报警记录数据层等软件功能可在存储与触摸稳定后分批接入。
