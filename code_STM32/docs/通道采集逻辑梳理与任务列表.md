## 一、现有驱动在干什么（先读懂这段）

### 1.1 用大白话说

板子上有一颗外部芯片 **AD7606**，一次能同时采 **8 路**模拟电压，每路结果是一个 **有符号 16 位整数（int16）**。

MCU（STM32）不负责「片内 ADC」，而是：

1. 用定时器 **TIM12** 周期性给 AD7606 一个「开始转换」脉冲（CONVST，脚 **PB15**）  
2. AD7606 忙完后，**BUSY** 脚（**PB14**）从高变低  
3. MCU 进中断，通过 **FSMC 总线**连续读 8 次，得到 `raw[0]~raw[7]`  
4. 这 8 个数就是通道 1~8 的原始采样值  

**不是 DMA，不是轮询 ADC 寄存器**，是：**定时触发 + BUSY 中断 + 总线读数**。

### 1.2 一次完整采样的时间线

```text
TIM12 PWM 跳一下  →  AD7606 开始转换
        …（很短时间）…
BUSY 从高变低     →  进 EXTI 中断
中断里连读 8 次   →  raw[0]=CH1 … raw[7]=CH8
        ├─ 存进「最新快照」s_latest_raw[]
        └─ 若正在采集：调用 SampleHook（给业务层用）
```

### 1.3 驱动文件里每个函数干什么

文件（均在主工程内）：
- `amp-vibe-analyzer/code_STM32/user/bsp/src/bsp_ad7606.c`
- `amp-vibe-analyzer/code_STM32/user/bsp/include/bsp_ad7606.h`

| 函数 | 什么时候用 | 干什么（人话） |
|------|------------|----------------|
| `bsp_InitAD7606` | 上电初始化 | 设过采样=关、量程、硬件复位；快照清零 |
| `AD7606_StartRecord(hz)` | 点 Start Acq | 复位芯片 → 按 hz 开 PWM → 开 BUSY 中断 → 标记「正在采」 |
| `AD7606_StopRecord` | 点 Stop | 停 PWM、关中断、标记「不在采」 |
| `AD7606_EnterAutoMode` | Start 内部调用 | 算 TIM12 周期，让采样率 ≈ hz（限制 100~20000） |
| `HAL_GPIO_EXTI_Callback` | BUSY 下降沿自动进 | 确认是 BUSY 脚且已为低 → 调 `ReadNowAdc` |
| `AD7606_ReadNowAdc` | 中断里 | FSMC 地址 `0x60200000` 连读 8 次 → 存快照 → 调 Hook |
| `AD7606_GetLatestRaw(out)` | 采集任务里 | **关中断拷贝**最新 8 路，给业务用（线程安全） |
| `AD7606_GetSampleCount` | 调试/验收 | 已经成功采了多少「组」（每组 8 通道） |
| `AD7606_SampleHook` | 每次采完 | 默认空；业务层 `acq_pipeline` 会重载，把振动路推进缓冲 |

### 1.4 读数地址（驱动里写死的）

```c
#define AD7606_RESULT()  (*(volatile int16_t *)0x60200000UL)
// 连读 8 次：第 1 次 = 通道1，第 8 次 = 通道8
```

约定（代码注释已写明）：

| 读的次序 | 数组下标 | 软件 channel_id |
|----------|----------|-----------------|
| 第 1 次 | raw[0] | 1 |
| 第 2 次 | raw[1] | 2 |
| … | … | … |
| 第 8 次 | raw[7] | 8 |

### 1.5 和「业务层」的分界（先知道有这层即可）

驱动只负责：**给出 raw[8]**。  
后面「哪路是振动、算峰峰值、画曲线」在 `acq_pipeline.c` / `acq_task.c`，**本阶段验收「能否读数」只看驱动层是否稳定出 raw**。

## 二、读数验收标准（最终要满足这些）

做完各阶段填空后，用下面标准判断「是否满足读取数据要求」：

| # | 要求 | 怎样算通过 |
|---|------|------------|
| R1 | 引脚与 CubeMX/代码一致 | BUSY/RESET/OS/RANGE/CONVST 脚位表填完且与 `main.h` 一致 |
| R2 | 能启动/停止采样 | Start 后 `GetSampleCount` 增加；Stop 后不再增加 |
| R3 | 能读到 8 路 int16 | `GetLatestRaw` 返回 true，且数组有值（非永远全 0，除非输入真是 0） |
| R4 | 通道次序可信 | 只给某一模拟输入加信号时，对应 `channel_id` 变化（需原理图/端子对照） |
| R5 | 采样率大致正确 | 默认约 1000 组/秒（允许误差）；或与设定 hz 成比例 |

**软件侧（R1~R3）我可以根据代码先判「具备读数能力」。**
