# code_STM32 项目说明文档

## 一、项目概述

本项目为 **SignalX 旋转设备电流与振动信号采集分析系统** 的嵌入式固件工程，基于 **STM32CubeIDE + GCC** 构建，**无 Keil 代码大小限制**。

仓库路径：`amp-vibe-analyzer/code_STM32`（独立维护；若与工作区 `code` / `code_STM32` 并行，业务 GUI 需按约定同步）。

| 项目 | 说明 |
|------|------|
| 芯片 | STM32F407VGT6（Cortex-M4，168MHz，1MB Flash，128KB RAM + 64KB CCMRAM） |
| 显示 | 7 寸 RA8875，800×480 电阻触摸 |
| ADC | AD7606（8 通道同步采样，FSMC） |
| 存储 | SD 卡（FatFs）：配置 / 校准 / 报警与实时数据等 |
| RTOS | FreeRTOS |
| GUI | LVGL v8.4 |
| IDE | STM32CubeIDE（含 arm-none-eabi-gcc） |
| 调试/烧录 | **ST-Link（SWD）** — 工程内 `code Debug.launch` 已配置 |

## 四、板端：编译、烧录、调试（ST-Link）

### 4.1 所需软件

| 软件 | 说明 |
|------|------|
| STM32CubeIDE | 打开本工程、编译、一键 Debug/Run |
| ST-Link 驱动 | 安装 CubeIDE 时通常已带；也可装 [STM32CubeProgrammer](https://www.st.com/en/development-tools/stm32cubeprog.html) |
| USB 线 | 连接 ST-Link 调试器到 PC |

工程调试配置：`code Debug.launch`

- Probe：**ST-Link**
- 接口：**SWD**
- 频率：约 **4000 kHz**
- 复位策略：`connect under reset`（欠复位连接，利于已跑飞的芯片重新连接）
- 下载文件：`Debug/code.elf`
- 默认在 `main` 处停下

### 4.2 打开工程

1. 启动 **STM32CubeIDE**
2. **File → Open Projects from File System…**（或 Import → Existing Projects into Workspace）
3. 选择目录：  
   `e:\workspace\Signal_STM32\amp-vibe-analyzer\code_STM32`
4. 确认 Project Explorer 中出现工程（工程名多为 `code`）

**Windows 下若 Build 报 `sh.exe` Error 87：**

- 右键工程 → **Properties** → **C/C++ Build** → **Environment**
- 新增：`SHELL` = `cmd.exe`
- Apply → 再 Build

### 4.3 编译

1. 工具栏选择配置 **Debug**（默认）
2. **Project → Build Project**，或快捷键 **Ctrl+B**
3. 成功后产物：
   - `Debug/code.elf` — 调试/烧录用
   - `Debug/code.bin` / `code.hex` — 视设置而定，可供 CubeProgrammer 单独烧录

控制台无 `Error`、出现 `Finished building` / `Build Finished` 即表示通过。

### 4.4 ST-Link 接线（SWD）

本工程使用 **SWD**（不是全功能 JTAG）。MCU 固定调试脚：

| MCU 信号 | 引脚 | 说明 |
|----------|------|------|
| **SWDIO** | **PA13** | 双向数据 |
| **SWCLK** | **PA14** | 时钟 |
| **NRST** | **NRST** | 复位（强烈建议接，配合 connect under reset） |
| **GND** | GND | 与 ST-Link 共地（必须） |
| **VTref / 3V3** | 板子 3.3V | 给调试器作目标电平参考；**一般不靠 ST-Link 给整板供电** |

#### 最小接线（推荐 5 线）

```
ST-Link          目标板（STM32F407）
────────         ──────────────────
SWDIO    ─────►  PA13 (SWDIO)
SWCLK    ─────►  PA14 (SWCLK)
GND      ─────►  GND
3.3V/TVCC─────►  3.3V（仅作参考，确认电平一致）
NRST     ─────►  NRST
```

#### 常见 ST-Link 硬件形态

**1）ST-Link/V2（或兼容板）侧面排针（常见丝印）**

不同克隆板丝印可能对调，**以板上英文标注为准**，典型对应：

| ST-Link 标注 | 接到板子 |
|--------------|----------|
| SWIM / SWDIO / DIO | PA13 |
| SWCLK / CLK | PA14 |
| GND | GND |
| 3.3V | 3.3V（参考） |
| RST / NRST | NRST |

**2）标准 20-pin ARM JTAG/SWD 排针（部分底板）**

仅用 SWD 时常用脚（针脚编号以 Cortex-M 20-pin 约定为准）：

| 针脚 | 信号 |
|------|------|
| 1 | VTref（3.3V） |
| 7 | SWDIO |
| 9 | SWCLK |
| 15 | NRST |
| 4 / 6 / 8 / 10 / 12 / 14 / 16 / 18 / 20 | GND（至少接一根） |

**接线注意：**

1. **先共地，再供电，最后插 USB**（或板子已供电后再连 ST-Link）
2. 板子供电与 ST-Link 的 3.3V **不要两路灌电冲突**；整板应由板载电源供电，ST-Link 的 3.3V 多作 VTref
3. 杜邦线尽量短；接触不良时表现为 “No target detected”
4. 不要占用 PA13/PA14 做普通 GPIO（Cube 里 SYS 应保持 Serial Wire）

### 4.5 烧录与调试（CubeIDE）

1. 板子供电，ST-Link USB 接入 PC，确认设备管理器中有 **ST-Link**
2. 在 CubeIDE 中打开 **Run → Debug Configurations…**，确认选中 **code Debug**（ST-Link / SWD）
3. 点击工具栏 **Debug**（小虫子图标），或 **F11**
   - 会自动：编译（若需要）→ 通过 ST-Link 下载 `Debug/code.elf` → 停在 `main`
4. 按 **F8（Resume）** 继续运行；可在源码行号左侧打断点
5. 若只需烧录运行、不调试：可用 **Run**（Ctrl+F11），同样经 ST-Link 下载后复位运行

**仅烧录、不用 IDE 调试时**（可选）：

1. 打开 **STM32CubeProgrammer**
2. 连接方式选 **ST-Link**，Mode **Normal**，接口 **SWD**
3. Connect → Open file 选择 `Debug/code.elf` 或 `.bin` → Download
4. 成功后复位或重新上电

### 4.6 常见问题

| 现象 | 处理 |
|------|------|
| No STM32 target found | 查 GND/SWDIO/SWCLK；试接 NRST；板子是否上电；降低 SWD 频率 |
| 能连接但下载失败 | 检查是否写保护；换 USB 口；更新 ST-Link 固件（CubeProgrammer → Firmware upgrade） |
| 连接后立刻掉线 | 确认 PA13/PA14 未被改成普通 IO；用 connect under reset |
| 烧录成功但白屏/无界面 | 先确认 LCD/触摸供电与 FSMC；再查 FreeRTOS 任务是否跑起来 |

## 五、板端测试方法（建议顺序）

### 5.1 上电冒烟

1. 按第四节完成编译与烧录
2. Resume 后观察：
   - 背光/液晶是否出图
   - 顶部 Tab：通道设置 / 采集设置 / 主监控 / 报警记录
3. 无卡时 SD 相关功能可能失败，但 GUI 应仍能起来（视启动逻辑）

### 5.2 触摸与校准

1. 若无有效校准文件，启动可能先进入校准流程（`GUI_TOUCH_CALIB_MODE` 等见 `config.h`）
2. 校准结果写入 **SD**：`Config/touch_calib.bin`（不再依赖片内 Flash 空闲扇区）
3. 校准后点按 Tab、按钮，确认落点与显示一致
4. 详细步骤见：`docs/STM32+RA8875内置触摸ADC方案五点校准实操指南.md`

### 5.3 界面与弹窗

1. 通道设置：改名称、类型、单位，确认弹窗可正常关闭（自绘 `GUI_Dialog`，非 `lv_msgbox`）
2. 采集设置：改参数后 Apply
3. 主监控：Start/Stop，看曲线是否刷新；点状态区看报警状态弹窗

### 5.4 采集链路（需 AD7606 硬件）

1. 主监控点 Start
2. CubeIDE 中断点或表达式查看 `Acq_GetFrameCount()` 是否约 50 次/秒增长
3. 可对照 `docs/八通道采集与主监控显示工作任务.md` 中的板端检查项

### 5.5 SD 卡

1. 插入已格式化的 SD（FatFs 可识别的 FAT32）
2. 校准保存后确认卡上有 `Config/touch_calib.bin`
3. 通道/采集保存后确认 `Config/channel_config.json`、`Config/acq_settings.json`

## 七、采集与主监控模块（acq）

八通道 AD7606 采集、校准路由、主监控实时曲线。详见 `docs/八通道采集与主监控显示工作任务.md`。

| 模块 | 路径 | 职责 |
|------|------|------|
| BSP | `user/bsp/bsp_ad7606.*` | 8 路 raw、采样钩子 |
| 管线 | `acq_pipeline.c` | 校准、通用路由、振动环、PP、包络 |
| 任务 | `acq_task.c` | AcqTask、`Acq_Start/Stop` |
| 报警 | `alarm_manager.c` | 报警状态机与 GUI 联动 |
| GUI | `GUI_PostRealtimeData` | AcqTask → `lv_async_call` → chart |

**线程模型**：`guiTask` → `lv_timer_handler`；`acqTask` **不**直接操作 `lv_chart`。

## 十、注意事项

1. 不要手改 `Core/`、`Drivers/`（CubeMX 再生会覆盖）
2. 业务代码若与其它副本工程并行，注意 `inlcude` ↔ `include` 路径差异
3. 板端 LVGL：`Middlewares/Third_Party/LVGL/lv_conf.h`
4. CCMRAM 64KB 用于 LVGL 堆；主 SRAM 含显示条带缓冲
5. **调试器以 ST-Link 为准**；文档旧版写的 J-Link 已废弃
